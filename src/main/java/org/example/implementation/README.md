# 구현/시뮬레이션

## 소개

구현(Implementation)은 문제에서 요구하는 동작을 그대로 코드로 옮기는 유형이다. 특별한 알고리즘 지식 없이 문제 해석력과 코딩 정확도가 핵심이다.

시뮬레이션(Simulation)은 구현의 하위 유형으로, 주어진 규칙대로 상태를 단계별로 변화시키는 문제다. "로봇이 이동한다", "배열을 회전한다" 같은 게 대표적이다.

취업 코테에서 출제 빈도가 높다. 알고리즘 몰라도 풀 수 있어서 쉬워 보이지만, 실수 하나로 틀리기 쉬운 유형이다.

---

## 알고리즘 특징

### 자주 나오는 유형

- 2차원 배열 조작 (회전, 이동, 복사)
- 좌표 이동 (상하좌우, 방향 전환)
- 규칙 기반 상태 변화 (턴제 게임, 로봇 동작)
- 문자열 파싱 및 처리

### 공통 패턴

**방향 배열**

**범위 체크**


**상태 복사**

## 체크리스트

### 문제 풀기 전

- [ ] 입력과 출력을 한 줄로 정리했는가?
- [ ] 예제를 손으로 그려봤는가?
- [ ] 문제의 규칙을 수식이나 조건문으로 변환했는가?

### 구현 중

- [ ] 인덱스가 0부터인지 1부터인지 확인했는가?
- [ ] 배열 범위를 벗어나는 접근이 없는가?
- [ ] 방향 전환 시 순서가 맞는가? (시계/반시계)
- [ ] 원본 배열을 보존해야 하는데 덮어쓰고 있지 않은가?

### 제출 전

- [ ] 클래스명이 Main인가?
- [ ] package 선언을 삭제했는가?
- [ ] 예제 입력으로 테스트했는가?

## 풀이 기록

### BOJ 16967 - 배열 복원하기

- **핵심**: `B[i][j] = A[i][j] + A[i-X][j-Y]` 식을 `A[i][j] = B[i][j] - A[i-X][j-Y]`로 변형
- **막혔던 부분**: 문제 해석에 시간을 안 들이고 구현부터 들어가서 왔다갔다 함
- **배운 점**: 문제 이해에 시간 투자하는 게 결국 더 빠름. 예제를 손으로 그려보는 게 중요함
